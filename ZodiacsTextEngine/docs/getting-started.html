Getting Started

Prerequisites

To get started writing your story using the provided console application, you will need:

  • A text editor for writing your story.
  • A copy of the ZodiacsTextEngine console application.

If you intend to write custom functions or integrate it into another environment, you'll also need:

  • An IDE capable of creating .NET applications, such as Visual Studio, Visual Studio Code, or JetBrains Rider.
  • A copy of the ZodiacsTextEngine package.
  • A .NET project to integrate the ZodiacsTextEngine into.
For more information how to setup the engine in a .NET project, refer to Using ZodiacsTextEngine in a custom Application.

Writing Content

When using the provided console application, the game content files need to be placed in a folder named content next to the executable.

The ZodiacsTextEngine follows a file structure where each section is defined in its own file (called a "room"). The engine runs through the content of a room file sequentially, executing effects and displaying text until the end, where it awaits user input. Depending on user input, the story can branch into different rooms, execute other effects (e.g. adding an item to the player's inventory), or simply provide flavour text.

By default, the ZodiacsTextEngine uses the file named start.room as the starting point for the game.

The folder structure can look like this:

Game Directory
game.exe
content
start.room
entrance.room
hallway.room
dungeon
dungeon_entrance.room
dungeon_trap.room
...
The required file structure and name of the start room can be customized when writing a custom application.

Room files can be grouped into subfolders without restrictions, the subfolders have no effect on how the room files are processed.

See Room File Structure for detailed information on how to write room files.
