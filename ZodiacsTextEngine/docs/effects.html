Effects

A comprehensive list of all available effects in the ZodiacsTextEngine.

+TEXT

Writes text to the output window.

The text effect can be used to display single-line or multi-line text. It supports rich text tags which can change the color of the text or display the value of a variable.

Examples

  • //Single line text
  • +TEXT Hello, World!
  • //Multi-line text
  • +TEXT
  • This is a multi-line text effect.
  • It can span multiple lines.
  • It can also contain blank lines.
  • //Rich text tags
  • +TEXT
  • This text is <color=red>red<reset>.
  • This text has a <background=blue>blue background<reset>.
  • You have <var=hitpoints> left.
    •  Hello, World!
    This is a multi-line text effect.
    It can span multiple lines.

    It can also contain blank lines.
    This text is red.
    This text has a blue background.
    You have 73 hitpoints left. _

    +PLAIN_TEXT

    Writes text to the output window without any formatting.

    This effect works the same as +TEXT, but it does not process rich text tags. This allows you to display special characters that would otherwise be interpreted as tags.

    Examples

  • +PLAIN_TEXT Hello, World!
  • +PLAIN_TEXT
  • This effect is similar to +TEXT
  • but it does not process <color=yellow>rich text tags<reset>
  • +TEXT < Welcome, traveler! >
    •  Hello, World!
    This effect is similar to +TEXT
    but it does not process <color=yellow>rich text tags<reset>
    < Welcome, traveler! > _

    +SPACE

    Writes blank lines to the output.

    This effect can be used to create space between text elements in the output window.
    The number of blank lines can be specified as an argument, defaulting to 1 if not provided.

    Examples

  • +TEXT This is a line of text.
  • +SPACE
  • +TEXT This is another line of text.
  • +SPACE 3
  • +TEXT This line is separated by three blank lines.
    •  This is a line of text.

    This is another line of text.



    This line is separated by three blank lines. _

    +CLEAR

    Clears the output window.

    This effect removes all text from the output window. It is useful for resetting the display before showing new content.

    Examples

  • +TEXT Hello, World!
  • //Clear the output window
  • +CLEAR
  • //Display new content
  • +TEXT Welcome to the ZodiacsTextEngine!
    •  Welcome to the ZodiacsTextEngine! _

    +COLOR

    Changes the color of the text.

    This effect allows you to change the color of the text in the output window. It affects all subsequent text until the color is changed again or reset.
    This effect can be combined with the +BACKGROUND effect and rich text tags.
    The rich text tag <reset> defaults back to this color.

    Examples

  • //Example usage
  • +TEXT Normal text
  • +COLOR red
  • +TEXT This text is red.
  • +COLOR blue
  • +TEXT This text is blue.
  • +RESET
  • +TEXT This text is back to normal.
    •  Normal text
    This text is red.
    This text is blue.
    This text is back to normal. _

    +BACKGROUND

    Changes the background color of the text.

    This effect allows you to change the background color of the text in the output window. It affects all subsequent text until the background is changed again or reset.
    This effect can be combined with the +COLOR effect and rich text tags.
    The rich text tag <reset> defaults back to this color.

    Examples

  • //Example usage
  • +TEXT Normal text
  • +BACKGROUND red
  • +TEXT This text has a red background.
  • +BACKGROUND blue
  • +TEXT This text has a blue background.
  • +RESET
  • +TEXT This text is back to normal.
    •  Normal text
    This text has a red background.
    This text has a blue background.
    This text is back to normal. _

    +RESET

    Resets all text formatting to the default.

    This effect clears all styles, including text and background color, returning to the default text appearance.

    Examples

  • +COLOR yellow
  • +TEXT This text is yellow.
  • +BACKGROUND gray
  • +TEXT This text is yellow with a gray background.
  • +RESET
  • +TEXT This text is back to normal.
    •  This text is yellow.
    This text is yellow with a gray background.
    This text is back to normal. _

    +VAR_SET / +VAR_ADD

    Sets or adds to a variable.

    This effect allows you to modify the value of a variable.
    Variables can be used to store game state information, such as player inventory or health.
    They can be checked in IF/ELSE blocks to control the flow of the game.

    Examples

  • +VAR_SET coins 7
  • +TEXT You are in front of a pile of coins. You have <var=coins> coins.
  • +TEXT You take three coins.
  • +VAR_ADD coins 3
  • +TEXT You now have <var=coins> coins.
  • IF coins > 5
  • +TEXT You have enough coins to buy a potion (costs 5 coins).
  • +TEXT You buy a potion.
  • +VAR_ADD coins -5
  • +VAR_ADD potions 1
  • +TEXT You now have <var=coins> coins and <var=potions> potion(s).
  • ELSE
  • +TEXT You don't have enough coins to buy a potion.
  • ENDIF
    •  You are in front of a pile of coins. You have 7 coins.
    You take three coins.
    You now have 10 coins.
    You have enough coins to buy a potion (costs 5 coins).
    You buy a potion.
    You now have 5 coins and 1 potion(s). _

    +SVAR_SET / +SVAR_ADD

    Sets or adds to a string variable.

    This effect allows you to modify the value of a string variable.
    String variables can be used to store text information, such as player names or other inputs.
    They can be checked in IF/ELSE blocks to control the flow of the game.
    Content without double quotes gets interpreted as a variable name.
    NOTE: String variables and numeric variables are separate and cannot be used interchangeably.

    Examples

  • +TEXT What is your name?
  • +SVAR_SET name "Harry"
  • +TEXT Hello, <var=name>! And what is your full name?
  • +SVAR_ADD name "Potter"
  • +TEXT Nice to meet you, <var=name>!
  • IF name == "Harry Potter"
  • +TEXT You're a wizard, Harry!
  • ENDIF
    •  What is your name?
    Hello, Harry! And what is your full name?
    Nice to meet you, Harry Potter!
    You're a wizard, Harry! _
  • //Define variable house
  • +SVAR_SET house "A house"
  • //Set variable x to string "house"
  • +SVAR_SET x "house"
  • +TEXT <var=x>
  • //Set variable y to the value of variable named house
  • +SVAR_SET y house
  • +TEXT <var=y>
  • //String concatenation
  • +SVAR_SET description "giant"
  • +SVAR_SET z "A "
  • +SVAR_ADD z description
  • +SVAR_ADD z " house."
    •  house
    A house
    A giant house.
    _

    +GOTO

    Goes to the room with the given name.

    The game proceeds immediately to the specified room, any following effects from the current room are skipped.

    Examples

  • //house.txt
  • @ENTER
  • +TEXT
  • There is a door leading to the back of the house.
  • You can LOOK through the window or go through the DOOR.
  • What will you do?
  • //Choice that is executed when the user types 'DOOR'
  • >DOOR
  • +TEXT Curious, you open the door and step outside.
  • +GOTO garden.txt
  • >WINDOW
  • +TEXT You look through the window and see a beautiful garden.
    •
  • //garden.txt
  • @ENTER
  • +COLOR green
  • +TEXT
  • You enter a beautiful garden.
    •  There is a door leading to the back of the house.
    You can LOOK through the window or go through the DOOR.

    What will you do?
    DOOR Curious, you open the door and step outside.
    You enter a beautiful garden. _

    +RETURN

    Returns to the same room.

    This is equivalent to +GOTO (name of current room), causing the room's enter event to be triggered again.
    This effect must be used cautiously when used within an @ENTER event, as it can lead to an infinite loop if not handled properly.

    Examples

  • @ENTER
  • +TEXT You are in front of a pile of coins. You have <var=coins> coins.
  • >TAKE COIN
  • +TEXT You take a coin.
  • +VAR_ADD coins 1
  • +RETURN
    •  You are in front of a pile of coins. You have 0 coins.
    TAKE COIN You take a coin.
    You are in front of a pile of coins. You have 1 coins.
    TAKE COIN You take a coin.
    You are in front of a pile of coins. You have 2 coins.
    TAKE COIN You take a coin.
    You are in front of a pile of coins. You have 3 coins. _

    +GAMEOVER

    Triggers a game over state.

    This effect is used to indicate that the game has ended, either due to a player loss or reaching the end of the story.
    Depending on the behaviour of the text interface, the game may either restart, return to a main menu, or exit altogether.

    Examples

  • +TEXT You have escaped!
  • +GAMEOVER
    •  You have escaped!
    ---------
    Game Over
    ---------
    [Press any key to restart]

    +EXIT

    Closes the game window.

    This terminates the game process immediately. To end the game normally, it is recommended to use +GAMEOVER instead.
    The behaviour of the exit effect can be overriden using a custom text interface.

    Examples

  • +TEXT To quit the game, type "QUIT"
  • > QUIT
  • +EXIT
    •  To quit the game, type "QUIT"
    > QUIT

    +WAIT

    Waits for the user to press any key.

    This effect pauses further execution until the user presses a key. It is useful for allowing the player to read text before continuing.
    The prompt can be customized with a custom message, otherwise it will revert to the default message [Press any key to continue].

    Examples

  • +TEXT Hello, World!
  • +WAIT
  • +TEXT You pressed a key!
  • +WAIT Further user input required
  • +TEXT Welcome aboard.
    •  Hello, World!
    [Press any key to continue]
    You pressed a key!
    [Further user input required]
    Welcome aboard. _

    +DELAY

    Waits a specific amount of seconds before continuing.

    This effect pauses execution for a specified number of seconds. It is useful for creating timed pauses in the text flow.
    The delay can be specified in seconds, and it can be a decimal value for more precise timing.
    During the delay, no user input is accepted, and the output window remains unchanged.

    Examples

  • +DELAY 0.5
  • +TEXT Big Man goes live.
  • +DELAY 2
  • +TEXT He threatens to destroy the earth with his portable thermonuclear bomb or even several.
    •  Big Man goes live.
    He threatens to destroy the earth with his portable thermonuclear bomb or even several. _

    +FUNC

    Executes a custom C# function.

    Use this effect to invoke a custom function declared in the project.
    The function accepts any number of parameters, which are passed as individual strings.
    With custom functions, you can implement complex logic that is not covered by the built-in effects.

    Examples

  • //Room file
  • +FUNC math_add 1 1
  • +FUNC math_add 5 7
  • +FUNC math_add -5 32
    •
  • //C# file
  • AddFunction("math_add", parameters =>
  • {
  • int a = int.Parse(parameters[0]);
  • int b = int.Parse(parameters[1]);
  • int result = a + b;
  • TextEngine.Interface.Write($"Result of {a} + {b} = {result}");
  • });
    •  Result of 1 + 1 =: 2
    Result of 5 + 7 = 12
    Result of -5 + 32 = 27 _

    +TODO

    A comment indicating that something is work in progress.

    This effect can be used to mark a specific location that is work in progress. All TODOs will be reported at the start of the game when running in debug mode.
    During gameplay, this effect does nothing.

    Examples

  • //This will be reported every time when running the game with debug mode on
  • +TODO Write more text
    •

    +BREAKPOINT

    Triggers a breakpoint in the debugger.

    When running in a debugging environment, this effect invokes System.Diagnostics.Debugger.Break(), halting execution and allowing the developer to inspect the current state.
    The effect itself does not produce any output.

    Examples

  • //Triggers a breakpoint when reaching this line
  • +BREAKPOINT
