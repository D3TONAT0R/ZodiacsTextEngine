Variables

Variables are a powerful feature in the ZodiacsTextEngine allowing writers to create more dynamic stories or implement simple mechanics based on various conditions.

Variable Types

There are two types of variables, which are independent of each other:

Numeric Variables (VARs)

Numeric variables use the Integer type and can be used to represent anything quantifyable or a true/false value, e.g. amount of coins, number of locations visited or whether the player has a specific key in their inventory (by checking if the value is greater than zero). Numeric variables use the VAR abbreviation in room files.

String Variables (SVARs)

String variables use the String type and can represent any piece of text, such as the player name. String variables use the SVAR abbreviation in room files.

Defining Variables

Variables can be set using the +VAR_SET and +SVAR_SET or effect respectively.

//Set 'coins' to 5 +VAR_SET coins 5 //Set 'money' to the same value as 'coins' +VAR_SET money coins //Set 'name' to "Hans" (without the quotation marks) +SVAR_SET name "Hans"

Adding/Subtracting Variables

Variables can be added together using the +VAR_ADD and +SVAR_ADD effects.

//Set 'coins' to 2 +VAR_SET coins 2 //Add 3 to 'coins' +VAR_ADD coins 3 //'coins' is now 5 +VAR_SET cost 4 //Subtract 4 from 'coins' by adding the negative value of 'cost' +VAR_ADD coins -cost //Append " Jr." to 'name' +SVAR_ADD name " Jr."

The following restrictions apply when naming a variable:

  • Names may only contain letters, digits and underscores (a-z,0-9,_).
  • Names cannot start with a digit.
  • Names are case-insensitive, however using lowercase is recommended.

Conditions

Conditions can be used to check the state of variables and control the flow of the story.

The IF statement is used to start a conditional block, which is closed with either the ENDIF or ELSE statement. The ELSE statement is optional and can be used to define an alternative path if the condition is not met. Conditions can be nested.

Indentations are optional but recommended to improve readability.
//Check if 'coins' is greater than 0 IF VAR coins > 0 +TEXT You have some coins! ENDIF //Check if 'name' is "Hans" IF name IS "Hans" +TEXT Hello, Hans! ELSE +TEXT Hello, stranger! ENDIF

The operator used in the condition determines whether the input variables are interpreted as Integer or String types. Integer and String variables cannot be used interchangeably.

Strings must always be defined with quotation marks, otherwise they will be interpreted as variable names.

The following operators are available:

Integer Operators

Operator Description
> Greater than
< Less than
>= Greater than or equal to
<= Less than or equal to
== Equal to
!= Not equal to

String Operators

Operator Description
IS Equal to
NOT / ISNOT / !IS Not equal to
CONTAINS Contains string
!CONTAINS Does not contain string
STARTSWITH Starts with string
!STARTSWITH Does not start with string
ENDSWITH Ends with string
!ENDSWITH Does not end with string

All string comparisons are not case-sensitive be default. To force case-sensitivity, operators can be appended with _CS, e.g. IS_CS.
